# SPEC

## 1. 概述

### 1.1 产品目标
本产品面向“有一定英语词汇量，但不敢开始阅读英文书”的用户。核心目标是帮助用户在阅读前快速判断一本书是否适合当前水平，并找出最值得优先记忆的高频单词，用最少的时间成本进入阅读阶段。

### 1.2 核心价值
- 判断一本英文书是否适合当前用户阅读
- 基于用户已掌握词汇，筛出真正需要学习的词
- 生成可直接导入其他学习软件的 CSV 结果

## 2. MVP 范围

### 2.1 必须包含
- 用户注册与登录，采用邮箱 + 密码
- 上传 `epub` 书籍
- 选择已掌握词汇范围：`COCA 1000..15000`
- 上传个人词表 `TXT`
- 发起异步分析任务
- 查看分析结果
- 下载 3 类 CSV
- 查看历史记录 / 书架
- 个人词库管理：上传、查看、新增、删除

### 2.2 明确不做
- 不支持 `PDF`
- 不做在线阅读器
- 不做单词释义
- 不做多人协作
- 不做支付
- 不做第三方登录
- 不做邮箱验证

## 3. 用户与使用场景

### 3.1 目标用户
已有一定词汇基础，但对直接阅读英文原著有心理门槛，希望先明确“要不要读”和“读前该先背哪些词”的用户。

### 3.2 核心场景
用户上传一本英文书，系统结合 `COCA` 范围和用户个人词库，计算该书的阅读难度与待记忆词表，并输出 CSV 供后续导入 Anki 等工具。

## 4. 核心业务规则

### 4.1 输入限制
- 仅支持英文书籍
- 仅支持 `epub`
- 书籍去重按文件哈希判定，而不是文件名
- 已存在书籍不重复上传，系统提示已存在，并允许基于原书重新分析

### 4.2 已掌握词定义
已掌握词集合 = 用户在下拉框中选择的 `COCA 1000..15000` 词汇 + 用户上传到数据库中的个人词表，按并集处理。

### 4.3 分析结果
系统必须生成以下 3 类 CSV：
- 完整单词列表
- 待记忆单词列表
- 95% 覆盖待记忆单词列表

三类 CSV 都必须包含字段：
- `序号`
- `单词`
- `书籍出现频率`
- `COCA 排行`
- `原型`
- `是否已掌握`

### 4.4 95% 覆盖率定义
95% 覆盖核心词指：先计算“已掌握词”对全书总词频的当前覆盖率。

- 若当前覆盖率已经 `>= 95%`，则 95% 覆盖核心词数量为 `0`
- 若当前覆盖率 `< 95%`，则只在“待记忆词”中按书中出现频率从高到低补齐
- 直到 `(已掌握词频 + 新学习词频) / 全书总词频 >= 95%`

等价定义：
- 95% 覆盖核心词 = 让全书陌生词频占比降到 `5%` 以下所需的最小未知词集合

### 4.5 分析快照
每次分析都必须保存快照。即使用户后续修改个人词库，历史分析结果也不自动变化。

## 5. 阅读建议规则

结果页需要基于“达到 95% 覆盖所需的待记忆词数”给出阅读建议卡片。

- `Level 1 / 强推荐阅读 / 绿色`：待记忆词数 `<= 200`
  提示：非常适合阅读，几乎无压力，可自然习得。
- `Level 2 / 推荐阅读 + 学习 / 黄色`：待记忆词数 `201-799`
  提示：可以阅读，但建议先学习核心词表。
- `Level 3 / 不推荐直接阅读 / 红色`：待记忆词数 `>= 800`
  提示：当前不建议直接开始阅读，应先积累基础词汇。

## 6. 用户流程

1. 用户注册并登录
2. 上传 `epub`
3. 选择已掌握的 `COCA` 范围
4. 系统合并用户个人词库
5. 提交分析任务
6. 后端立即返回 `job id`
7. 前端轮询任务状态
8. 分析完成后进入结果页
9. 用户查看统计结果、阅读建议并下载 CSV
10. 用户可在后续分析中复用原书文件重新分析

## 7. 页面需求

### 7.1 首页
- 上传书籍入口
- 选择 `COCA` 范围
- 引导登录 / 注册
- 产品价值说明

### 7.2 结果页
- 展示总词数
- 展示唯一词数
- 展示待记忆词数
- 展示 95% 覆盖所需词数
- 展示阅读建议卡片
- 提供 3 类 CSV 下载按钮

结果页不展示单词预览列表，交由用户自行决定如何使用导出结果。

### 7.3 个人词库页
- 上传 `TXT`
- 查看词表
- 新增单词
- 删除单词

底层需支持多份词表，但 MVP 前端先按“一份主词库”方式展示。

### 7.4 书架 / 历史记录页
- 书名
- 上传时间
- 分析状态
- 使用的 `COCA` 范围
- 待记忆词数量
- 重新分析入口
- 删除历史记录入口

## 8. 异步任务规范

### 8.1 状态
- `pending`
- `processing`
- `completed`
- `failed`
- `canceled`

### 8.2 对外接口策略
对外采用单次提交分析接口；对内拆为多阶段流水线：
- 上传 / 识别书籍
- 建立任务
- 解析书籍文本
- 提取单词并统计频率
- 匹配原型与 `COCA` 排名
- 合并已掌握词集合
- 生成 3 类结果
- 导出 CSV

### 8.3 错误展示
MVP 前端至少需要展示以下失败原因：
- 文件格式不支持
- 文件解析失败

## 9. 数据与存储

### 9.1 文件存储
原始书籍、用户上传词表、生成的 CSV 不长期存放在代码仓库中，应存放在服务器文件存储或对象存储中。

### 9.2 数据库存储
数据库保存：
- 用户信息
- 用户词库信息
- 书籍元数据
- 分析任务状态
- 分析快照
- 结果统计信息
- 文件存储路径或文件标识

## 10. 核心实体草案

- `users`
- `user_vocabularies`
- `user_vocabulary_items`
- `books`
- `analysis_jobs`
- `analysis_results`
- `analysis_result_items`

说明：
- `books` 用于保存去重后的书籍主记录
- `analysis_jobs` 支持同一本书重复分析
- `analysis_results` 保存一次分析快照
- `analysis_result_items` 保存词级结果明细

## 11. API 草案

- `POST /auth/register`
- `POST /auth/login`
- `POST /books/upload`
- `POST /analysis/jobs`
- `GET /analysis/jobs/{job_id}`
- `GET /analysis/results/{result_id}`
- `GET /analysis/results/{result_id}/downloads/{type}`
- `GET /books/history`
- `DELETE /analysis/results/{result_id}`
- `POST /vocabularies/upload`
- `GET /vocabularies`
- `POST /vocabularies/items`
- `DELETE /vocabularies/items/{item_id}`

其中 `downloads/{type}` 的 `type` 至少支持：
- `all_words`
- `to_memorize`
- `coverage_95`

## 12. 删除规则

删除历史分析记录时：
- 删除展示记录
- 删除结果文件
- 删除相关任务与结果元数据
- 仅当某书无任何关联分析任务时，才允许进一步删除书籍主记录

## 13. 非功能要求

- 上传接口应在数秒内返回 `job id`，不得阻塞页面
- 普通英文小说的分析目标时间为 `30-120 秒`
- 前端必须可正确处理轮询、失败态和重试提示
- 后端需记录任务日志，便于排查解析失败
- 仅允许访问用户自己的书籍、词库和分析结果
- 文件路径与存储标识不得直接暴露内部实现细节

## 14. 技术方向

- 后端：`FastAPI`
- 前端：`Nuxt`
- 注释可使用中文
- 变量名、接口字段名、模型名使用英文

## 15. 当前不纳入范围

以下内容不属于本版 `SPEC`：
- `PDF` 支持
- 第三方词典接入
- 单词释义
- 在线阅读器
- 支付能力
